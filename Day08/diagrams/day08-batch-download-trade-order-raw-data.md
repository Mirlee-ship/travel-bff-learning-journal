\# Day08：订单分片导出调用链



\## 完整调用链



```mermaid

flowchart TD

    A\["后台订单页面"] --> B\["用户选择筛选条件"]

    B --> C\["用户点击导出"]

    C --> D\["保留列表筛选条件"]

    D --> E\["请求 batchDownloadTradeOrderRawData"]

    E --> F\["进入 biz/index.ts 统一入口"]

    F --> G\["路由到 trade-bff / web"]

    G --> H\["读取 pageIndex 和 pageSize"]

    H --> I\["查询当前批次订单"]

    I --> J\["补充并转换导出字段"]

    J --> K\["生成 headerMap 和 data"]

    K --> L\["返回当前批次数据"]

    L --> M{"是否还有下一批"}

    M -->|是| N\["pageIndex 加一"]

    N --> E

    M -->|否| O\["结束数据收集"]

    O --> P\["生成或保存最终导出结果"]

```



\## 简化版调用链



```text

后台选择订单条件

→ 点击导出

→ 保留列表筛选条件

→ 请求当前批次

→ pageIndex + pageSize 分片查询

→ 转换导出字段

→ 返回 headerMap + data

→ 判断是否还有下一批

→ 继续查询或结束

```



\## 面试讲解重点



1\. 页面分页用于展示当前页，导出分片用于获取完整结果。

2\. 同一次导出中，筛选条件保持不变，主要递增 `pageIndex`。

3\. 分片可以降低单次查询、响应体和内存压力。

4\. `headerMap` 表示导出列定义，`data` 表示当前批次数据。

5\. 一批失败时具备按批重新处理的条件。
