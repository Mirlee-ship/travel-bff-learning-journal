\# Day 3：商品检索接口调用链图



\## searchItem 完整调用链



```mermaid

flowchart TD

&#x20;   A\["旅行社商品页面提交搜索条件"] --> B\["POST /commodity/search-item"]

&#x20;   B --> C\["执行 searchItem"]

&#x20;   C --> D{"是否传入 tagId"}

&#x20;   D -- "是" --> E\["查询标签关联的商品 ID"]

&#x20;   D -- "否" --> F{"是否传入 keyword"}

&#x20;   E --> F

&#x20;   F -- "是" --> G\["调用 commonSearchItem 获取候选商品 ID"]

&#x20;   F -- "否" --> H\["继续构造查询条件"]

&#x20;   G --> I{"候选商品 ID 是否为空"}

&#x20;   I -- "是" --> J\["提前返回 data 空数组和 total 0"]

&#x20;   I -- "否" --> H

&#x20;   H --> K\["组装 conditions 和 queryParams"]

&#x20;   K --> L\["构建列表 SQL 和总数 SQL"]

&#x20;   L --> M\["添加排序和分页"]

&#x20;   M --> N\["Promise.all 并行查询列表和总数"]

&#x20;   N --> O{"当前页是否有商品"}

&#x20;   O -- "否" --> P\["返回 data 空数组和 total"]

&#x20;   O -- "是" --> Q\["收集当前页商品 ID"]

&#x20;   Q --> R\["批量查询商品头图"]

&#x20;   R --> S\["批量查询关联城市"]

&#x20;   S --> T\["解析标签并计算利润"]

&#x20;   T --> U\["组装页面商品模型"]

&#x20;   U --> V\["返回 data 和 total"]

```



\## 简化记忆



```text

关键词或标签

→ 候选商品 ID

→ 动态筛选

→ 排序分页

→ 列表和总数并行查询

→ 批量补充头图和城市

→ 返回 data + total

```



\## 核心职责



```text

文本搜索：

确定候选商品范围



结构化筛选：

价格、结算价、天数和来源



分页查询：

当前页列表和总数



数据聚合：

头图、城市、标签和利润

```
